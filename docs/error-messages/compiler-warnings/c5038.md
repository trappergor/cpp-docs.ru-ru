# <a name="compiler-warning-c5038"></a>Предупреждение компилятора C5038

> данные-член "*член1*«будет инициализирован после элемента данных»*член2*«член данных»*член*«будет инициализирован после базовым классом»*base_class* '

Члены класса инициализируются в порядке их объявления, а не порядке их отображения в списках инициализаторов. Это предупреждение означает, что порядок инициализации не является таким же, как порядок объявления членов данных, а также базовых классов. Это может привести к поведению не определен во время выполнения, если инициализация одного члена в списке зависит от инициализации элемента, который объявлен в более поздней версии.

Это предупреждение появился в Visual Studio 2017 версии 15.3 и отключен по умолчанию. Используйте [/Wall](../../build/reference/compiler-option-warning-level.md) включить все предупреждения, отключенные по умолчанию или __/w__*n*__5038__ для включения C5038 как уровень *n* предупреждение. Дополнительные сведения см. в разделе [компилятора предупреждения, являются отключено по умолчанию](../../preprocessor/compiler-warnings-that-are-off-by-default.md). Сведения о том, как отключить предупреждения компилятора версией, см. в разделе [предупреждения компилятора по версиям компилятора](compiler-warnings-by-compiler-version.md).

## <a name="example"></a>Пример

В следующем примере Visual Studio 2017 версии 15.3 (с/Wall) выдает «предупреждение C5038: элемент данных «A::y» будет инициализирован после элемента данных «A::x»»:

```cpp
struct A
{
    A(int a) : y(a), x(y) {} // C5938 Initialized in reverse, y reused
    int x;
    int y;
};
```

Чтобы устранить эту проблему, упорядочить список инициализатора должен иметь тот же порядок, что и объявления. Похожее предупреждение возникает, когда один инициализатор или оба ссылаются на члены базового класса.
