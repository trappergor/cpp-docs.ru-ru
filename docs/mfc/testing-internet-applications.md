# <a name="testing-internet-applications"></a>Тестирование интернет-приложений

Существуют некоторые уникальные трудности тестирования в Интернете, особенно для приложений, выполняющихся на веб-сервере. Первоначального тестирования, возможно, выполняются с помощью однопользовательском режиме клиент, подключающийся к на тестовом сервере. Это будет полезно для отладки кода.

Вы также хотите протестировать в реальных условиях: от нескольких клиентов, подключенных через высокоскоростные подключения, а также низким высокоскоростных последовательных линий, включая модем подключения. Может затруднить для имитации реальных условиях, но определенно стоит расходов на время разработки возможные сценарии и их выполнении. Если это возможно будет также можно использовать средства, чтобы сделать ресурсы и нагрузочное тестирование. Определенные классы ошибок, таких как время ошибок, трудно найти, так и для последующего воспроизведения.

Одной из проблем программирования Интернет является видимым. Множество обращений к веб-узла может замедлить работу сервера. Сервер корректно снижать производительность. Вы хотите запретить все, что может быть уничтожения данных на компьютере пользователя, при сбое приложения (например, повреждение данных при записи в реестр или при записи файлов cookie на стороне клиента).
