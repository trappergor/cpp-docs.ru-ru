# <a name="sequence-of-operations-for-building-mfc-applications"></a>Последовательность операций для сборки приложений MFC

В следующей таблице описаны общие последовательности, обычно может следовать при разработке приложения MFC.

### <a name="sequence-for-building-an-application-with-the-framework"></a>Последовательность для построения приложений с помощью платформы

|Задача|Вы|Платформы|
|----------|------------|------------------------|
|Создайте схему приложения.|Запустите [мастер приложений MFC](../mfc/reference/mfc-application-wizard.md). Укажите нужные параметры на страницах параметров. Варианты, что делает приложение COM-компонент, контейнер или того и другого; Добавление службы автоматизации; и обеспечение поддержки базы данных приложения.|Мастер приложений MFC создает файлы скелет приложения, включая исходные файлы для вашего приложения, документа, представления и окна фрейма; файл ресурсов. файл проекта; и другие, все в соответствии с требованиями.|
|См. в разделе, какие платформы и мастер приложений MFC предложения без добавления строки кода.|Создать общую схему приложения и запустите его в Visual C++.|Запуск скелет приложения наследует многие стандартные **файл**, **изменить**, **представление**, и **помочь** команд меню из .NET framework. Для приложения MDI вы также получаете полнофункциональный меню Windows и среда управляет создания, расположение и деструкция дочерних окон интерфейса MDI.|
|Создание пользовательского интерфейса приложения.|С помощью Visual C++ [редакторы ресурсов](../windows/resource-editors.md) для визуального редактирования интерфейса пользователя приложения:<br /><br /> -Создайте меню.<br />-Задать ускорители.<br />-Создание диалоговых окон.<br />-Создание и редактирование точечные рисунки, значки и курсоры.<br />-Edit панели инструментов, создан автоматически с помощью мастера приложений MFC.<br />-Создание и изменение других ресурсов.<br /><br /> Также можно проверить диалоговые окна в редактор диалоговых окон.|Файл ресурса по умолчанию, созданный с помощью мастера приложений MFC предоставляет многие из необходимых ресурсов. Visual C++ позволяет редактировать существующие ресурсы и добавлять новые ресурсы, легко и визуально.|
|Сопоставьте функции обработчика меню.|Используйте **события** кнопку [окно "Свойства"](/visualstudio/ide/reference/properties-window) соединиться с меню и сочетания клавиш функции обработчика в коде.|Окно свойств вставляет записей карты и пустая функция шаблоны в исходные файлы, укажите и управляет много вручную задач кодирования.|
|Запишите свой код обработчика.|Используйте представление классов для перехода непосредственно к коду в редакторе исходного кода. Введите код для функций-обработчиков. Дополнительные сведения об использовании представления классов и о мастерах, добавьте код в проект, см. в разделе [Добавление функциональных возможностей с помощью мастеров кода](../ide/adding-functionality-with-code-wizards-cpp.md).|Представление классов открывает редактор, отобразится пустая функция шаблона и помещает курсор автоматически.|
|Сопоставьте кнопки панели инструментов команд.|Сопоставить каждой кнопки на панели инструментов команду меню или сочетание клавиш, назначив кнопки идентификатор соответствующие команды.|Платформа управляет рисунка, включение, отключение, проверки и других визуальных характеристик элемента кнопки панели инструментов.|
|Тестирование функций обработчика.|Перестройте программу и использовать встроенные средства отладки для проверки правильной работы обработчиков.|Можно шаг или трассировку кода, чтобы увидеть, способ вызова обработчиков. Если вы заполнили код обработчика, обработчики выполнения команд. Платформа автоматически отключит пункты меню и кнопки панели инструментов, которые не обрабатываются.|
|Добавить [диалоговым окнам](../mfc/dialog-boxes.md).|Ресурсы шаблона диалогового окна для разработки приложений с помощью редактора диалоговых окон. Затем создание класса диалогового окна и код, который обрабатывает диалоговое окно.|Платформа управляет диалоговом окне, а также упрощает получение сведений о, введенный пользователем.|
|Инициализировать, проверки и извлечения данных диалогового окна.|Можно также определить, каким образом элементы управления диалогового окна, инициализировать и проверять. Используйте Visual Studio, чтобы добавить переменные-члены в класс диалоговых окон и сопоставить их с элементов управления диалоговых окон. Укажите правила проверки для применения к каждому элементу управления, который пользователь вводит данные. При желании, предоставляют свои собственные собственные атрибуты проверки.|Среда управляет инициализации диалогового окна и проверки. При вводе недопустимых данных, платформа отображает окно сообщения и пользователь может повторно ввести данные.|
|Создайте дополнительные классы.|Представление классов можно используйте для создания дополнительных документов, представления и классы окна фрейма, помимо тех, которые созданы с помощью мастера приложений MFC. Можно создать классы набора записей дополнительную базу данных, классы диалоговых окон и т. д. (В представлении класса можно создавать классы, не являющиеся производными классами MFC.)|Представление классов добавляет эти классы исходные файлы и помогает определить свои соединения для любых команд, которые они обрабатывают.|
|Добавьте компоненты готовые к использованию в приложении.|Используйте `New Item dialog box` для добавления различных элементов.|Эти элементы можно легко интегрировать в приложение и сэкономить немало работы.|
|Реализуйте класс документа.|Реализация класса документов от приложения или классов. Добавьте переменные-члены для хранения структур данных. Добавление функции-члены в качестве интерфейса к данным.|Платформа уже знает, как взаимодействовать с файлами данных документа. Его можно открыть и закрыть файлы документов, чтение и запись данных документа и обработки других пользовательских интерфейсах. Вы можете сосредоточиться на способ управления этим состоянием данных документа.|
|Функция открытия, сохранения и сохраните как команды.|Написание кода для документа `Serialize` функция-член.|Платформа отображает диалоговые окна для **откройте**, **Сохранить**, и **Сохранить как** команды **файл** меню. Она записывает и операций чтения обратно на документ, используя формат данных, указанный в вашей `Serialize` функция-член.|
|Реализация класса представления.|Реализуйте один или несколько классов представления, соответствующие документы. Реализация функции-члены этого представления, которые сопоставлены с пользовательского интерфейса с помощью представления классов. Широкий набор [CView](../mfc/reference/cview-class.md)-производные классы, в том числе [CListView](../mfc/reference/clistview-class.md) и [CTreeView](../mfc/reference/ctreeview-class.md).|Платформа управляет большей частью связи между документа и его представлением. Функции-члены этого представления получить доступ к документу представления для отображения своего изображения на экран или печать страницы и обновлять структуры данных документа в ответ на команды редактирования пользователем.|
|Улучшения печати по умолчанию.|Если необходима поддержка многостраничной печати, переопределяет представления функций-членов.|Платформа поддерживает **печати**, **параметры страницы**, и **Предварительный просмотр печати** команды **файл** меню. Необходимо указать его как разбить документ на несколько страниц.|
|Добавьте прокрутки.|Если требуется для поддержки прокрутки, являются производными класса представления или классы из [CScrollView](../mfc/reference/cscrollview-class.md).|Представление автоматически добавляет полосы прокрутки при окна становится слишком мал.|
|Создание представления форм.|Если будет производиться представлений ресурсов шаблона диалогового окна, являются производными класса представления или классы из [CFormView](../mfc/reference/cformview-class.md).|Представление использует ресурса шаблона диалогового окна для отображения элементов управления. Пользователь может перейти от элемента управления к элементу управления в представлении.|
|Создание форм баз данных.|Приложения доступа к данным на основе форм, создать производный от класса представления [CRecordView](../mfc/reference/crecordview-class.md) (для программирования с использованием ODBC).|Представление работает по принципу представление формы, но ее элементов управления подключены к полям [CRecordset](../mfc/reference/crecordset-class.md) объект, представляющий таблицу базы данных. MFC перемещает данные между элементами управления и набор записей для вас.|
|Создает простой текстовый редактор.|Если требуется настроить для представления можно воспользоваться простым текстовым редактором, являются производными представления класса или классов из [CEditView](../mfc/reference/ceditview-class.md) или [CRichEditView](../mfc/reference/cricheditview-class.md).|Представление будет содержать редактирования функции, поддержка буфера обмена и файлового ввода вывода. `CRichEditView` предоставляет стиль текста.|
|Добавление окна разделителей.|Если вы хотите поддерживать Разделение окна, добавление [CSplitterWnd](../mfc/reference/csplitterwnd-class.md) объекта окна области SDI или дочернего окна интерфейса MDI и связать его в окне [OnCreateClient](../mfc/reference/cframewnd-class.md#oncreateclient) функция-член.|Платформа предоставляет элементы управления splitter рядом с полосами прокрутки и управляет разделение вам заглянуть в несколько областей. Если пользователь разделяет окна, платформа создает и присоединяет объекты дополнительное представление для документа.|
|Сборки, тестирования и отладки приложения.|Используйте средства Visual C++ для создания, тестирования и отладки приложения.|Visual C++ позволяет настраивать компиляции, ссылки и другие параметры. Он также позволяет просматривать структуру кода и класс источника.|
