# <a name="c6236"></a>C6236

> предупреждение C6236: ( \<expression> &#124;&#124; \<non-zero constant> ) всегда является ненулевой константой

Это предупреждение означает, что в правой части логической операции или, которая выполняется в контексте теста, обнаружено ненулевое значение константы, отличное от единицы. Логически это означает, что тест является избыточным и может быть безопасно удален. Кроме того, предполагается, что программист может использовать другой оператор, например, оператор равенства ( `==` ), побитового и ( `&` ) или побитового оператора XOR ( `^` ) для проверки на наличие определенного значения или флага.

Это предупреждение не создается для общей идиомы, если ненулевая константа равна 1, так как она используется для выборочного включения путей кода во время компиляции. Однако предупреждение создается, если ненулевая константа формируется выражением, результатом которого является 1, например 1 + 0.

## <a name="example"></a>Пример

Этот код показывает, как может выглядеть предупреждение C6236. Поскольку `INPUT_TYPE` значение не равно 0, выражение `n || INPUT_TYPE` всегда имеет ненулевое значение, и **`else`** предложение никогда не выполняется. Однако `INPUT_TYPE` является константой со значением, отличным от 1, которое предполагает, что оно предназначено для сравнения:

```cpp
#define INPUT_TYPE 2
#include <stdio.h>

void f( int n )
{
   if ( n || INPUT_TYPE ) // analysis warning C6236
   {
      puts( "Always gets here" );
   }
   else
   {
      puts( "Never enters here" );
   }
}
```

В следующем коде вместо этого используется оператор побитового и ( `&` ) для проверки, `INPUT_TYPE` задан ли бит входного параметра `n` :

```cpp
#define INPUT_TYPE 2
#include <stdio.h>

void f( int n )
{
   if ( n & INPUT_TYPE ) // no warning
   {
      puts( "Bitwise-AND comparison is true" );
   }
   else
   {
      puts( "Bitwise-AND comparison is false" );
   }
}
```
